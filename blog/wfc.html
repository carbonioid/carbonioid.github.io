in which i make a bad WFC implementation

I'm doing a lot of research into procedural generation (terrain generation, cellular automata, all that) currently, so I thought I may as well some put something here for each algorithm I implement. Here's one for Wavefunction Collapse.

some examples of a better implementation

What is it?

Wavefunction Collapse (also known as WFC or more generally Model Synthesis) is similar to a satisfiability/SAT solver: you give it a set of constraints (you could think of these as boolean variables) and it tries to generate a solution that satisfies them all. The special thing about WFC is that it can determine what those constraints are from an example input.

Despite the fancy name, it's probably an algorithm that you've used at least once in your life. In Robert Heaton's article on WFC (which I would recommend) they give the analogy of making a seating plan, which I think is quite helpful.

To describe it informally, you start with all seats blank (in superposition) and then choose some seat and assign it a random value. Then look at your constraints and see what that means about the surrounding seats (maybe Bob won't sit next to Alice but Charlie has to.) Then choose the seat with the least remaining possibilities and choose a value for it. Repeat until you either have a full seating plan or a seat where no-one could possibly sit.

I like how this reflects the way one would actually solve this problem: if you start at one seat you'll probably assign a seat near it next, not one 50 seats away; and that's the same with WFC because the seats with the least possibilities are the ones nearest to the seats with people already in them (as opposed to wavefunctions sitting all over the place.) It also makes me feel better about solving problems like this because now I have a fancy name to point to!

What is it, actually?

Informal definition aside, here's how you use it in 2D terrain generation: split your terrain into a tesselation of square tiles. Each one borders its four neighbours. Then we choose a couple tile types: maybe land, sea and coast.

each square borders its neighbours like this, which you probably got already but I want to make images

Rules

So now we need to know how the squares can border each other. This is where our example input comes in. Basically, you analyse each square to see what it borders. Write down every possible combination (e.g. sea and border land to the north, coast can border sea to the east) and how often it occurs. Once you've done this you get a list of frequencies which you could think of as a matrix if you were so inclined.

I am so inclined (it's really a 3-dimensional matrix with a 2D slice for each tile type, but i digress)

In the cases where the frequency is 0.0, this interaction never happens and we should definitely not generate it when making our new image.

Generating the Image

In the same way as the seating plan, set the wavefunctions of all the tiles to every possible tile type. Then choose a tile (I use the top left corner) and set it to a random tile. Now, repeat these steps:

  • Step 1: look really hard at every tile on the board (you could be clever and flood fill out from the changed tile, but I don't.) If any of the surrounding tiles have a definite value, check what tiles can border that value in the relevant direction. If any of the tiles in the tile's wavefunction can't border the definite tile, remove them.
  • Step 2: look even harder at every tile without a collapsed wavefunction and find the one with the lowest entropy. I've seen entropy calculated in a few of different ways (e.g. Shannon entropy), but I just take it as the number of possibilities (which is bad.) If you can't find any tiles without a collapsed waveform, you're finished and can stop.
  • Step 3: collapse the wavefunction of this tile. For each possibility in its wavefunction (and only these possibilities), look up the weight of that tile type in that direction for each of its four neighbours. Add these up for each tile type and they become the weights you use to randomly choose the state of this tile. You just added one more definite tile to your grid!

This is a very unoptimized implementation of the function and I should probably be using shannon entropy, but it's still fun to play around with it. Here's an output on the left along with the input on the right.

Because red never borders itself, the code has worked out that is forms dividing borders between grey and green! Admittedly, it's a shame that it still generates those random red dots everywhere. This simple implementation isn't an amazing algorithm but it does have a fun name and it's a pretty fun thing to code if you have an afternoon free.

Further Reading

As an aside, while reading around it turns out that you can think of WFC rulesets like Markov chains. An implementation where tiles can influence at most 1 tile away (which is what I made) is equivalent to an order-1 Markov chain. More generally, one where tiles can influence n tiles away is equivalent to an order-n Markov chain.

Also, there exists an implementation by Shaad Alaka and Rafael Bidarra called 'Hierarchical Semantic wave function collapse' (that's the name of the relevant paper too) where tiles can take on multiple hierarchical states during generation: a tile might be a castle tile and on the next iteration either become a wall or courtyard tile. Check that out if you're interested.
